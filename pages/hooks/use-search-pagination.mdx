# useSearchPagination

`useSearchPagination` provides a small, reusable state model for search + pagination list experiences.

## What It Handles

- current `PaginationParams` state
- query updates through `handleSearch`
- page reset to `1` on every new search
- full param replacement through `handlePaginationChange`

## Usage

```tsx
import { Button, Page, Text, useSearchPagination } from '@rapidset/rapidkit';

export function UsersListState() {
  const { paginationParams, handleSearch, handlePaginationChange } =
    useSearchPagination();

  return (
    <Page
      onSearch={handleSearch}
      searchPlaceholder="Search users"
      className="max-h-none"
    >
      <div className="space-y-4 rounded-xl border border-border bg-card p-4">
        <div className="flex items-center justify-between gap-3">
          <Text
            as="small"
            tone="muted"
            text={
              'Page ' +
              String(paginationParams.page ?? 1) +
              ' / Size ' +
              String(paginationParams.size ?? 20)
            }
          />

          <Button
            label="Next"
            onClick={() => {
              handlePaginationChange({
                ...paginationParams,
                page: (paginationParams.page ?? 1) + 1,
              });
            }}
          />
        </div>
      </div>
    </Page>
  );
}
```
